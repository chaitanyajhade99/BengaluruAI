# BengaluruAI — Event-Driven Congestion Intelligence

**Flipkart Gridlock Hackathon 2.0 · Theme 2: Event-Driven Congestion (Planned & Unplanned)**

---

## What this does

BengaluruAI predicts the impact of traffic events in Bengaluru **before** officers are deployed. Given an event cause, corridor, and time, it outputs:

- Road closure probability (LightGBM classifier, AUC 0.776)
- Predicted incident duration (XGBoost regressor, R² 0.529)
- Alert tier (CRITICAL / HIGH / MEDIUM / LOW)
- Recommended officers, barricade sets, diversion activation
- Post-event learning loop that improves forecasts over time

## Instructions to run

### Option 1 — Open the dashboard directly (no install needed)

Open `dashboard/index.html` in any modern browser. The dashboard is fully self-contained.

- **Overview tab**: hourly patterns, event cause breakdown, closure rates, corridor risk bars
- **Predict Event tab**: configure any event and get instant AI forecast + recommendations
- **Hotspot Map tab**: interactive Leaflet.js map of Bengaluru's top congestion junctions
- **Post-Event Learning tab**: correction factor table and accuracy trend charts

### Option 2 — Retrain the ML models

```bash
# 1. Install dependencies
pip install lightgbm xgboost scikit-learn pandas numpy

# 2. Train (takes ~60 seconds)
cd gridlock_project
python src/pipeline.py

# This generates models/ files and prints:
# [Classifier] F1=0.43  AUC=0.776
# [Regressor]  RMSE(log)=1.845  R²=0.529  MedianAE=49 min
```

### Option 3 — Run a custom prediction from Python

```python
import pickle, json
from src.pipeline import load_and_clean, build_features, predict_event, PostEventLearner

# Load models
with open('models/classifier.pkl', 'rb') as f:
    clf_bundle = pickle.load(f)
with open('models/regressor.pkl', 'rb') as f:
    reg = pickle.load(f)
with open('models/features.json') as f:
    features = json.load(f)

# Load lookup tables into module
import src.pipeline as p
with open('models/lookup_tables.json') as f:
    lut = json.load(f)
p._CAUSE_STATS = lut['cause_stats']
p._CORRIDOR_STATS = lut['corridor_stats']

learner = PostEventLearner()

result = predict_event(
    clf=clf_bundle['model'],
    reg=reg,
    features=features,
    learner=learner,
    event_cause='public_event',
    corridor='Bellary Road 1',
    event_type='planned',
    start_datetime_IST='2024-03-15 18:00',
    latitude=12.9716,
    longitude=77.5946,
    priority='High',
)
print(result)
# {'alert_tier': 'HIGH', 'closure_probability_pct': 62.1, 
#  'predicted_duration_hrs': 2.4, 'officers_recommended': 6, ...}
```

## Model details

### Feature engineering (32 features)
| Category | Features |
|---|---|
| Temporal | hour_IST, is_peak_morning, is_peak_evening, day_of_week, month, is_weekend, is_night |
| Event | is_planned, priority_high, cause_ordinal, cause_closure_rate, cause_severity_score |
| Vehicle | veh_type_encoded, has_vehicle |
| Spatial | latitude, longitude, corridor_risk, corridor_closure_rate, junction_risk, zone_density, dist_from_centre |
| Composite | impact_index, peak_x_severity, planned_x_cause |

### Model A — Closure classifier (LightGBM)
- Target: `requires_road_closure` (binary)
- Class imbalance: 11.1:1 (negative:positive) → `scale_pos_weight=11.1`
- Best threshold: 0.20 (F1-optimised)
- **AUC-ROC: 0.776  |  F1: 0.43  |  Accuracy: 91%**

### Model B — Duration regressor (XGBoost)
- Target: `log(duration_minutes)` — log-transform handles extreme skew
- **R²: 0.529  |  RMSE (log scale): 1.845  |  Median AE: 49 minutes**

## Post-event learning

After each event resolves, the system compares `predicted_duration` vs `actual_duration` and logs the delta per key `(cause, corridor, peak/off-peak)`. Future predictions for the same bucket are corrected by the running mean error. This is visible in the **Post-Event Learning** dashboard tab.
